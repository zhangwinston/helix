# Quick Start: IME自动控制功能

**Date**: 2024-12-19  
**Feature**: IME自动控制功能

## 功能概述

IME自动控制功能根据编辑模式和光标位置自动开启或关闭输入法（IME），提升中文用户在代码编辑时的体验。

## 核心行为

### 基本工作流程

1. **View初始化**: 编辑器打开时，IME自动关闭
2. **进入Insert模式**: 
   - 如果光标位于字符串/注释区域 → 恢复之前保存的IME状态
   - 如果光标位于代码区域 → 保持IME关闭
3. **光标移动**: 
   - 从代码区域移动到字符串/注释区域 → 不保存当前状态，根据之前保存的状态恢复IME（如果之前保存的是开启则开启，否则保持关闭）
   - 从字符串/注释区域移动到代码区域 → 保存当前IME状态并关闭IME
4. **退出Insert模式**: 如果当前IME状态是开启则保存状态，如果关闭则不保存；然后关闭IME

### IME敏感区域

- **敏感区域**（IME可以开启）:
  - 字符串内容（不包括前导引号 `"`）
  - 注释内容（不包括首部符号 `//`, `/*`）
  - 字符串尾部引号的第一个字符
  - 注释尾部符号的第一个字符（如 `*/` 的 `*`）

- **非敏感区域**（IME应该关闭）:
  - 代码区域
  - 字符串前导引号
  - 注释首部符号

### 特殊场景

- **无法语法解析的文件**: 整个文件视为敏感区域
- **语法解析延迟**: 解析完成前，整个文件视为敏感区域
- **多光标**: 根据主光标（最后一个活动光标）位置决定IME状态
- **手动切换IME**: 如果光标位置和模式未改变，保持用户手动设置的IME状态

## 使用示例

### 示例1: 代码编辑

```rust
fn main() {
    let message = "你好";  // ← 光标在这里时，IME自动开启
    println!("{}", message);  // ← 光标在这里时，IME自动关闭
}
```

### 示例2: 模式切换

1. 在字符串区域开启IME输入中文
2. 按 `Esc` 退出Insert模式 → IME开启状态被保存，然后关闭IME
3. 按 `i` 重新进入Insert模式，光标仍在字符串区域 → IME自动恢复开启状态

**注意**: 如果退出Insert模式时IME是关闭状态，则不会保存这个关闭状态。

### 示例3: 多光标编辑

```rust
let a = "hello";  // ← 光标1
let b = "world";  // ← 光标2（主光标）
```

IME状态根据主光标（光标2）的位置决定。

## 错误处理

- IME API调用失败时，功能静默失败并记录日志
- 用户可以继续手动控制IME
- 编辑流程不受影响

## 性能特性

- IME状态切换响应时间 < 100ms
- 对编辑器性能影响 < 5%
- 仅在Insert模式下进行检测，其他模式无开销

## 技术实现要点

1. **事件驱动**: 使用 `OnModeSwitch` 和 `SelectionDidChange` 事件
2. **语法解析**: 利用tree-sitter检测字符串/注释区域
3. **状态管理**: 每个view独立维护IME状态
4. **平台抽象**: 通过trait抽象不同操作系统的IME API

## 测试场景

1. 在代码区域和字符串区域之间移动光标，验证IME自动切换
2. 模式切换时验证IME状态保存和恢复
3. 多光标场景验证主光标逻辑
4. 语法解析失败时验证降级行为
5. IME API调用失败时验证错误处理
